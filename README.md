# Документация проекта Social Network

## 1. Описание проекта

**Social Network** — веб-приложение социальной сети, построенное на **React + Vite** с использованием **Supabase** в качестве backend-as-a-service.  
Приложение включает:

- авторизацию и регистрацию;
- ленту постов;
- лайки и комментарии;
- поиск и управление друзьями;
- редактирование профиля;
- личное хранилище файлов;
- защиту данных на уровне Supabase RLS-политик.

Также в проекте присутствует файл `main.py` с примером отдельного **FastAPI**-backend'а на SQLite/SQLAlchemy. Он не используется фронтендом напрямую, но может служить заготовкой для альтернативной серверной реализации.

---

### Frontend

### Backend / Data layer

## 3. Функциональность

### 3.1 Авторизация
Пользователь может:
- зарегистрироваться по email/password;
- войти в существующий аккаунт;
- выйти из системы.

При регистрации в `signUp()` передаются метаданные:
- `username`
- `full_name`

Профиль создаётся автоматически триггером в базе данных.

### 3.2 Лента
Пользователь может:
- просматривать все посты;
- создавать пост;
- ставить и убирать лайк;
- оставлять комментарии;
- удалять свои посты;
- удалять свои комментарии.

### 3.3 Друзья
Пользователь может:
- искать других пользователей;
- отправлять заявку в друзья;
- принимать входящие заявки;
- отклонять заявки;
- удалять друга.

### 3.4 Профиль
Пользователь может:
- просматривать свой профиль;
- редактировать:
  - username,
  - full_name,
  - bio,
  - avatar_url;
- видеть статистику:
  - количество постов,
  - количество друзей,
  - количество лайков.

### 3.5 Мои файлы
Пользователь может:
- загружать файлы;
- перетаскивать файлы в область загрузки;
- скачивать собственные файлы;
- удалять собственные файлы.

Поддерживаются:
- `.doc`
- `.docx`
- `.pdf`
- `.xls`
- `.xlsx`
- `.ppt`
- `.pptx`
- `.zip`

Максимальный размер файла: **100 МБ**.

## 5. Установка и запуск

### 5.1 Установка зависимостей
```bash
npm install
```

### 5.2 Переменные окружения
Создайте файл `.env` в корне проекта:

```env
VITE_SUPABASE_URL=your_supabase_project_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
VITE_STORAGE_BUCKET=private-files
```

Если `VITE_STORAGE_BUCKET` не указан, по умолчанию используется `private-files`.

### 5.3 Запуск в режиме разработки
```bash
npm run dev
```

После запуска Vite обычно доступен по адресу:

```text
http://localhost:5173
```

### 5.4 Сборка проекта
```bash
npm run build
```

### 5.5 Предпросмотр собранной версии
```bash
npm run preview
```

## 9. API и модель данных

## 9.1 Важное уточнение по API
Фронтенд **не использует собственный REST API** для основной работы.  
Вместо этого он обращается напрямую к Supabase через `@supabase/supabase-js`.

То есть фактический API проекта состоит из:
1. операций Supabase Auth;
2. запросов к таблицам PostgreSQL;
3. операций Supabase Storage;
4. RLS-политик, ограничивающих доступ.

---

## 9.2 Supabase Auth API

### Регистрация
Используется:
```js
supabase.auth.signUp({
  email,
  password,
  options: {
    data: { username, full_name }
  }
})
```

### Вход
Используется:
```js
supabase.auth.signInWithPassword({
  email,
  password
})
```

### Выход
Используется:
```js
supabase.auth.signOut()
```

### Получение сессии
Используется:
```js
supabase.auth.getSession()
```

### Подписка на изменения состояния
Используется:
```js
supabase.auth.onAuthStateChange(...)
```

## 9.3 Таблица `profiles`

### Назначение
Профиль пользователя.

### Поля
- `id` — uuid, PK, ссылка на `auth.users.id`
- `username` — уникальный логин
- `full_name` — полное имя
- `avatar_url` — URL аватара
- `bio` — описание профиля
- `created_at`
- `updated_at`

### Операции в приложении
- чтение профиля текущего пользователя;
- чтение профилей всех пользователей для поиска друзей;
- обновление своего профиля.

### Пример запроса
```js
supabase.from('profiles').select('*').eq('id', userId).maybeSingle()
```

---

## 9.4 Таблица `posts`

### Назначение
Публикации пользователей.

### Поля
- `id`
- `user_id`
- `content`
- `image_url`
- `created_at`
- `updated_at`

### Операции
- получить все посты;
- создать пост;
- редактировать свой пост;
- удалить свой пост.

### Пример создания
```js
supabase.from('posts').insert([{ user_id: user.id, content: newPost }])
```

### Пример выборки с отношениями
```js
supabase.from('posts').select(`
  *,
  profiles:user_id (username, full_name, avatar_url),
  likes (id, user_id),
  comments (
    id,
    content,
    created_at,
    profiles:user_id (username, full_name, avatar_url)
  )
`)
```

### Назначение

### Путь


### Политики

- чтение только своей папки;
- удаление только своей папки.

### Получение файла
Для скачивания используется signed URL с коротким временем жизни.

## 10.1 `profiles`
- чтение: всем аутентифицированным пользователям;
- обновление: только владелец;
- вставка: только владелец.

## 10.2 `posts`
- чтение: всем аутентифицированным;
- создание: только от своего имени;
- обновление: только владелец;
- удаление: только владелец.

## 10.3 `comments`
- чтение: всем аутентифицированным;
- создание: только от своего имени;
- удаление: только владелец.

## 10.4 `likes`
- чтение: всем аутентифицированным;
- создание: только от своего имени;
- удаление: только владелец.

## 12. FastAPI-прототип в `main.py`

Файл `main.py` содержит отдельную серверную реализацию на FastAPI + SQLAlchemy + SQLite.  
Это не часть основного фронтенд-потока, но документировать его полезно.

### Технологии
- FastAPI
- SQLite
- SQLAlchemy
- Pydantic
- CORS для `http://localhost:5173`

### Модели
- `Profile`
- `Post`
- `Comment`
- `Like`

### Реализованные эндпоинты

#### `GET /api/posts`
Возвращает список постов.

#### `POST /api/posts`
Создаёт пост.

Параметры:
- тело запроса: `PostCreate`
- query-параметр: `user_id`

#### `DELETE /api/posts/{post_id}`
Удаляет пост текущего пользователя.

Параметры:
- `post_id`
- `user_id`

#### `POST /api/posts/{post_id}/like`
Переключает лайк:
- если лайк уже есть — удаляет;
- если лайка нет — создаёт.

Параметры:
- `post_id`
- `user_id`

### Примечание
В текущем виде этот backend выглядит как демонстрационный или экспериментальный слой.  
Основное приложение в React использует Supabase напрямую, а не этот FastAPI-сервер.

## 14. Ограничения и особенности

- Приложение требует настроенный Supabase-проект.
- Без корректных `.env` переменных фронтенд не запустится.
- Пользовательские операции защищены RLS.
- Файлы приватны и доступны только владельцу.
- `main.py` не связан с React автоматически — это отдельная реализация backend API.
- В Supabase должна быть создана таблица `profiles` и настроен триггер автосоздания профиля.
- Для хранения файлов нужен bucket `private-files`.
